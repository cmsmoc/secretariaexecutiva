Mini CMS
Baseado em Planilha
Arquitetura completa de um sistema de gerenciamento de conteúdo leve, sem backend tradicional. Google Sheets como banco de dados, GAS como API, e HTML/JS estático como apresentação — com controle total via planilha.

Princípio central

O Google Sheets é o único local de edição de conteúdo. Nenhum acesso ao código é necessário para alterar textos, publicar notícias, ativar módulos ou customizar cores. O GAS converte os dados em JSON e o JavaScript do site renderiza os módulos dinamicamente.

O que o editor faz na planilha
  • Adiciona uma linha em NOTICIAS com status=publicado
  • Muda a ordem dos módulos em HOME_LAYOUT
  • Ativa/desativa um popup em POPUP
  • Muda a cor principal em THEME
  • Cadastra uma nova ata em ATAS
O que acontece automaticamente
  • GAS filtra apenas status=publicado
  • JSON é regenerado na próxima requisição
  • Site lê o JSON novo via fetch()
  • CSS variables são aplicadas dinamicamente
  • Módulos renderizam na ordem configurada
02 · Fluxo

Fluxo de Dados

Do editor à tela do cidadão em 4 etapas.

01
Editor preenche Sheets
Gestor edita células. Nenhum código. Interface familiar do Google.
→
02
GAS processa
Apps Script lê as abas, filtra, ordena e serializa em JSON estruturado.
→
03
JSON exposto via Web App
GAS publica um endpoint HTTPS público. O site faz fetch() neste URL.
→
04
JS renderiza módulos
O front-end lê o JSON, instancia templates e injeta no DOM em ordem.
Cache estratégico

O GAS usa CacheService para armazenar o JSON por 5 minutos. Isso evita leitura repetida da planilha a cada requisição, respeitando os limites de quota do Google.

04 · Google Apps Script

Camada de Dados

O GAS é o único "backend" do sistema. Um Web App deployado que lê as Sheets, processa e devolve JSON. A função doGet() é o endpoint principal.

Code.gs — Função principal
// ── CONSTANTES DE ABAS ──────────────────────────────────────
const ABA = {
  CONFIG:    'CONFIG',
  THEME:     'THEME',
  LAYOUT:    'HOME_LAYOUT',
  MENU:      'MENU',
  NOTICIAS:  'NOTICIAS',
  ATAS:      'ATAS',
  REUNIOES:  'REUNIOES',
  DOCUMENTOS:'DOCUMENTOS',
  POPUP:     'POPUP',
  BANNERS:   'BANNERS',
  ACADEMIA:  'ACADEMIA_CONSELHEIRO',
  EVENTOS:   'EVENTOS',
};

// ── CACHE ────────────────────────────────────────────────────
const CACHE_KEY = 'cms_json_v1';
const CACHE_TTL = 300; // segundos (5 min)

// ── ENDPOINT PRINCIPAL ────────────────────────────────────────
function doGet(e) {
  const action = e.parameter.action || 'full';

  // Retorno CORS + JSON
  const output = ContentService
    .createTextOutput(getPayload(action))
    .setMimeType(ContentService.MimeType.JSON);

  return output;
}

// ── PAYLOAD COM CACHE ─────────────────────────────────────────
function getPayload(action) {
  const cache = CacheService.getScriptCache();
  const cacheKey = `${CACHE_KEY}_${action}`;
  const cached = cache.get(cacheKey);

  if (cached) return cached;

  const ss = SpreadsheetApp.getActiveSpreadsheet();
  let data;

  switch(action) {
    case 'full':
      data = buildFullPayload(ss); break;
    case 'noticias':
      data = readSheet(ss, ABA.NOTICIAS, filterPublicado); break;
    default:
      data = { error: 'action inválida' };
  }

  const json = JSON.stringify(data);
  cache.put(cacheKey, json, CACHE_TTL);
  return json;
}

// ── BUILD PAYLOAD COMPLETO ────────────────────────────────────
function buildFullPayload(ss) {
  const hoje = new Date();

  return {
    _gerado_em: hoje.toISOString(),
    config:     readConfig(ss),
    theme:      readTheme(ss),
    menu:       readSheet(ss, ABA.MENU, filterAtivo, sortOrdem),
    home_layout:readSheet(ss, ABA.LAYOUT, filterAtivo, sortOrdem),
    noticias:   readSheet(ss, ABA.NOTICIAS, filterPublicado, sortDataDesc),
    atas:       readSheet(ss, ABA.ATAS, filterPublicado, sortDataDesc),
    reunioes:   readSheet(ss, ABA.REUNIOES, filterFuturas, sortDataAsc),
    documentos: readSheet(ss, ABA.DOCUMENTOS, filterPublicado, sortDataDesc),
    popup:      readPopup(ss, hoje),
    banners:    readBanners(ss, hoje),
    academia:   readSheet(ss, ABA.ACADEMIA, filterPublicado, sortDataDesc),
  };
}

// ── LEITURA GENÉRICA DE ABA ───────────────────────────────────
function readSheet(ss, nomeAba, filterFn, sortFn) {
  const sheet = ss.getSheetByName(nomeAba);
  if (!sheet) return [];

  const [headers, ...rows] = sheet.getDataRange().getValues();
  let data = rows
    .filter(r => r.some(c => c !== '')) // remove linhas vazias
    .map(row => {
      const obj = {};
      headers.forEach((h, i) => {
        const key = h.toString().toLowerCase().trim();
        obj[key] = _normalize(row[i]);
      });
      return obj;
    });

  if (filterFn) data = data.filter(filterFn);
  if (sortFn)   data = data.sort(sortFn);
  return data;
}

// ── CONFIG: key→valor em objeto ──────────────────────────────
function readConfig(ss) {
  const rows = readSheet(ss, ABA.CONFIG);
  return Object.fromEntries(rows.map(r => [r.chave, r.valor]));
}

// ── THEME: array de tokens ──────────────────────────────────
function readTheme(ss) {
  return readSheet(ss, ABA.THEME);
}

// ── POPUP com filtro de data ─────────────────────────────────
function readPopup(ss, hoje) {
  return readSheet(ss, ABA.POPUP, row => {
    if (!row.ativo) return false;
    if (row.data_inicio && new Date(row.data_inicio) > hoje) return false;
    if (row.data_fim   && new Date(row.data_fim)   < hoje) return false;
    return true;
  }, (a, b) => a.prioridade - b.prioridade);
}

// ── BANNERS com expiração ───────────────────────────────────
function readBanners(ss, hoje) {
  return readSheet(ss, ABA.BANNERS, row => {
    if (!row.ativo) return false;
    if (row.data_expiracao && new Date(row.data_expiracao) < hoje) return false;
    return true;
  });
}

// ── NORMALIZAÇÃO de células ──────────────────────────────────
function _normalize(val) {
  if (val instanceof Date) return val.toISOString().split('T')[0];
  if (val === 'TRUE'  || val === true)  return true;
  if (val === 'FALSE' || val === false) return false;
  return val;
}

// ── FILTROS ──────────────────────────────────────────────────
const filterAtivo     = r => r.ativo === true;
const filterPublicado  = r => r.status === 'publicado';
const filterFuturas    = r => r.status === 'agendada';

// ── ORDENAÇÕES ───────────────────────────────────────────────
const sortOrdem    = (a,b) => (a.ordem||99) - (b.ordem||99);
const sortDataDesc = (a,b) => new Date(b.data_publicacao||b.data) - new Date(a.data_publicacao||a.data);
const sortDataAsc  = (a,b) => new Date(a.data) - new Date(b.data);

// ── INVALIDAR CACHE (chamar ao salvar Sheets) ─────────────────
function invalidarCache() {
  CacheService.getScriptCache().removeAll([
    'cms_json_v1_full', 'cms_json_v1_noticias'
  ]);
}

js/api.js — Fetch com cache local
// Cache local em memória por sessão
let _cacheLocal = null;

async function fetchCMS() {
  if (_cacheLocal) return _cacheLocal;

  // Lê endpoint do meta tag (configurável por ambiente)
  const endpoint = document.querySelector('meta[name="gas-endpoint"]')?.content;
  if (!endpoint) throw new Error('gas-endpoint não configurado');

  const res = await fetch(`${endpoint}?action=full`);
  if (!res.ok) throw new Error(`Erro HTTP: ${res.status}`);

  _cacheLocal = await res.json();
  return _cacheLocal;
}

// Retorna dados de uma aba específica
async function getData(chave) {
  const cms = await fetchCMS();
  return cms[chave] || [];
}
js/main.js — Orquestrador
import { fetchCMS } from './api.js';
import { aplicarTema } from './theme.js';
import { renderizarPopup } from './popup.js';
import * as Templates from './modules/index.js';

async function init() {
  const cms = await fetchCMS();

  // 1. Aplicar tema CSS
  aplicarTema(cms.theme);

  // 2. Renderizar menu
  renderizarMenu(cms.menu);

  // 3. Renderizar módulos em ordem
  const main = document.getElementById('cms-main');
  const modulos = cms.home_layout.filter(m => m.ativo);

  for (const modulo of modulos) {
    const dados = cms[modulo.fonte_dados?.toLowerCase()] || [];
    const limite = modulo.limite || 10;
    const templateFn = Templates[modulo.template]; // ex: Templates['feed-cards']

    if (!templateFn) { console.warn(`Template não encontrado: ${modulo.template}`); continue; }

    const html = templateFn({
      modulo,
      dados: dados.slice(0, limite),
      config: cms.config
    });

    const section = document.createElement('section');
    section.setAttribute('data-modulo', modulo.modulo_id);
    section.innerHTML = html;
    main.appendChild(section);
  }

  // 4. Popup (se houver)
  renderizarPopup(cms.popup);
}

document.addEventListener('DOMContentLoaded', init);

// Template: feed-cards — usado para NOTICIAS
export function feedCards({ modulo, dados }) {
  const titulo = modulo.titulo_override || 'Últimas Atualizações';

  const catMap = {
    resolucao:   { label: 'Resolução',  cls: 'tag-green' },
    reuniao:     { label: 'Reunião',    cls: 'tag-blue'  },
    conferencia: { label: 'Conferência',cls: 'tag-yellow'},
    informe:     { label: 'Informe',    cls: 'tag-navy'  },
  };

  const cards = dados.map(n => {
    const cat = catMap[n.categoria] || { label: n.categoria, cls: 'tag-navy' };
    const dt  = new Date(n.data_publicacao).toLocaleDateString('pt-BR');
    return `
      <article class="feed-item">
        <div class="feed-date">${dt}</div>
        <div class="feed-body">
          <span class="tag ${cat.cls}">${cat.label}</span>
          <h3><a href="/noticia/${n.slug}">${n.titulo}</a></h3>
          <p>${n.resumo || ''}</p>
        </div>
      </article>
    `;
  }).join('');

  return `
    <div class="section-wrap fundo-${modulo.fundo || 'off-white'}">
      <div class="section-header">
        <h2 class="section-title">${titulo}</h2>
        <a href="/noticias" class="more-link">Ver todas →</a>
      </div>
      <div class="feed">${cards}</div>
    </div>
  `;
}

08 · Customização

Sistema de Tema

O THEME do Sheets alimenta as CSS variables do site. Trocar uma cor na planilha reflete no site sem tocar no código.

js/theme.js — Aplicação dinâmica
export function aplicarTema(tokens) {
  if (!tokens?.length) return;
  const root = document.documentElement;
  tokens.forEach(t => {
    if (t.token && t.valor) root.style.setProperty(t.token, t.valor);
  });
}

/* Exemplo: THEME sheet tem linha:
   token = --color-primary | valor = #1B6CB5
   → document.documentElement.style.setProperty('--color-primary', '#1B6CB5')
   → Toda propriedade CSS que usar var(--color-primary) é afetada */
Tokens de tema recomendados
  • --color-primary · --color-secondary · --color-accent · --color-text
  • --color-bg · --color-surface · --color-border
  • --font-display (ex: 'Montserrat', sans-serif)
  • --font-body (ex: 'Open Sans', sans-serif)
  • --radius-sm · --radius-md · --radius-lg

Popup, Banner e Push

js/popup.js — Sistema de popup
export function renderizarPopup(popups) {
  if (!popups?.length) return;
  const popup = popups[0]; // maior prioridade

  // Controle de frequência via localStorage
  const key = `popup_${popup.id}_visto`;
  if (popup.frequencia === 'uma_vez' && localStorage.getItem(key)) return;
  if (popup.frequencia === 'por_sessao' && sessionStorage.getItem(key)) return;

  const el = document.createElement('div');
  el.id = 'cms-popup';
  el.innerHTML = `
    <div class="popup-overlay" onclick="this.parentElement.remove()"></div>
    <div class="popup-box" role="dialog" aria-modal="true">
      <button class="popup-close" onclick="document.getElementById('cms-popup').remove()"
              aria-label="Fechar">×</button>
      <h2 class="popup-title">${popup.titulo}</h2>
      <p class="popup-msg">${popup.mensagem}</p>
      ${popup.botao_url ? `
        <a href="${popup.botao_url}" class="popup-btn">${popup.botao_label}</a>
      ` : ''}
    </div>
  `;

  document.body.appendChild(el);

  // Registrar como visto
  if (popup.frequencia === 'uma_vez')   localStorage.setItem(key, '1');
  if (popup.frequencia === 'por_sessao') sessionStorage.setItem(key, '1');
}

Estratégia de push (OneSignal)

O GAS pode disparar notificações push via REST API do OneSignal quando uma notícia nova é publicada. Basta um trigger onEdit no GAS que detecta mudança de status → publicado na aba NOTICIAS e chama o endpoint de push com o título e URL da notícia.

10 · Governança

Padrões e Boas Práticas

Regras simples que garantem que o sistema funcione bem por anos, mesmo com múltiplos editores.

Nomenclatura de campos
  • Nomes de abas: SNAKE_CASE maiúsculo (HOME_LAYOUT)
  • Cabeçalhos de colunas: snake_case minúsculo (data_publicacao)
  • IDs: numéricos sequenciais, nunca reutilizar
  • Slugs: somente letras, números e hífens
  • Status: sempre letras minúsculas sem acento
Preenchimento seguro
  • Nunca apagar linhas — use status = arquivado
  • Datas sempre no formato AAAA-MM-DD
  • Booleanos: TRUE / FALSE (maiúsculo)
  • URLs de Drive: sempre compartilhamento público "Qualquer pessoa com o link"
  • Não usar células mescladas
Controle de acesso
  • Apenas editores designados com permissão de edição
  • Aba CONFIG: proteger com senha de intervalo
  • Visitantes externos: somente leitura
  • GAS: executar como "proprietário" somente
  • Auditar alterações via histórico de versões do Sheets
Crescimento e manutenção
  • Novas seções = nova aba + novo módulo JS
  • Novos campos = adicionar coluna no final (não no meio)
  • Invalidar cache após grandes atualizações
  • Versionar o GAS com comentários de data e autor
  • Backup mensal da planilha via Google Takeout
11 · Guia de Uso

Passo a Passo

Para o editor não técnico: o que fazer para cada tarefa comum.

Publicar uma notícia

1
Abrir a aba NOTICIAS
Acesse a planilha do CMS-MOC e clique na aba "NOTICIAS".
2
Adicionar nova linha no final
Nunca inserir no meio. Sempre na última linha disponível.
3
Preencher os campos obrigatórios
titulo, resumo, data_publicacao (AAAA-MM-DD), categoria e deixar status = rascunho.
4
Revisar e alterar status
Quando pronto para publicar, mudar status de "rascunho" para "publicado".
5
Aguardar até 5 minutos
O cache do GAS expira em 5 minutos. Após isso, o site exibe a notícia automaticamente. Para publicação imediata, chamar a função invalidarCache() no GAS.

Reordenar seções da home

1
Abrir HOME_LAYOUT
Cada linha corresponde a uma seção do site.
2
Editar a coluna "ordem"
Mudar o número. Ex: trocar "noticias" de ordem 3 para 2, e "reunioes" de 2 para 3. O GAS ordena automaticamente pelo menor número.
3
Ativar ou desativar
Para ocultar uma seção completamente, mudar ativo para FALSE. A seção some do site sem excluir dados.

Ativar um popup

1
Ir para a aba POPUP
Cada linha = um popup configurado.
2
Preencher título, mensagem e botão
Campos: titulo, mensagem, botao_label, botao_url.
3
Definir datas e frequência
data_inicio e data_fim (opcional). frequencia = uma_vez para mostrar só uma vez por navegador.
4
Ativar
Mudar ativo para TRUE. O popup de menor prioridade numérica é exibido.
Regra de ouro

Se algo parecer errado no site, o primeiro passo é verificar o JSON diretamente — abrir o endpoint do GAS no navegador e inspecionar o que está sendo retornado. 99% dos problemas são de preenchimento incorreto na planilha, não de código.
