Deploy & Manual
Sistema 11ª CMS

Este documento guia o deploy completo do novo sistema web baseado em API JSON — e serve como manual permanente de configuração e gestão para a equipe da Secretaria Executiva.

📁 Google Sheets como DB ⚙️ Apps Script REST API 🌐 GitHub Pages ⏱ ~30 min para deploy completo
Visão Geral

Como o sistema funciona

Três camadas independentes que se comunicam via JSON. Cada camada pode ser atualizada sem quebrar as outras.

📊

Camada 1 — Dados

Google Sheets funciona como banco de dados. 5 abas estruturadas: CONFIG, PRESENCA, DELEGADOS, AVISOS e FOTOS. Editável pela equipe sem código.

⚙️

Camada 2 — API Backend

Google Apps Script publica uma Web App que lê a planilha e retorna JSON puro. Toda regra de negócio fica aqui — o frontend não precisa saber nada.

🌐

Camada 3 — Frontend

HTML + CSS + JS hospedado no GitHub Pages. Consome a API via fetch(). Só renderiza dados — não contém lógica de negócio.

💡

Por que essa arquitetura é melhor?

A versão antiga consumia CSV diretamente via URL pública. Isso causava latência (~2–4s), falhas silenciosas de parsing e código frágil. Com a API JSON, os dados chegam estruturados, validados e com tratamento de erro padronizado.

Acessar a Planilha Google

Confirme que você tem acesso de editor à planilha central do sistema.

📊 https://docs.google.com/spreadsheets/d/1sEEW3rrUYFlHHUlhnR4u5sVNA3BOw-cN_8TY8FgoMLI
1
Abra a planilha no navegador
Clique no link acima. Você deve ter acesso de Editor. Se aparecer "Solicitar acesso", peça ao responsável pela conta Google da Secretaria Executiva.
2
Confirme que está logado com a conta correta
O sistema deve ser gerenciado pela conta oficial do CMS, não uma conta pessoal. Verifique no canto superior direito do Google.
⚠️ Se usar uma conta pessoal, a API ficará associada a essa conta. Use sempre a conta institucional da Secretaria Executiva.
Deploy · Passo 2 de 6

Instalar o código no Apps Script

O backend da API vive dentro da própria planilha, como um script vinculado.

1
Abra o editor do Apps Script
Na planilha, clique no menu Extensões → Apps Script. Uma nova aba abrirá com o editor de código.
2
Apague o código existente
No editor, você verá um arquivo chamado Código.gs ou Code.gs. Selecione tudo o que está nele (Ctrl+A) e delete.
3
Cole o código do backend
Abra o arquivo 11cms-api-backend.gs que foi gerado, copie todo o conteúdo e cole no editor. Salve com Ctrl+S.
✓ Após salvar, o editor deve mostrar o nome do projeto no topo. Se pedir para nomear o projeto, use "11CMS-API".
4
Verifique a função doGet
Role o código até encontrar function doGet(e). Se estiver lá, o roteador da API está instalado corretamente.
Deploy · Passo 3 de 6

Publicar a Web App (API pública)

Este passo transforma o Apps Script em uma URL de API acessível pelo frontend.

⚠️

Atenção: este passo já foi feito antes

A URL https://script.google.com/macros/s/AKfycbzPh29ncgWef.../exec já está implantada. Se o código não foi alterado, você não precisa reimplantar. Só refaça se trocou a conta Google ou se quer atualizar o código da API.

1
No Apps Script, clique em "Implantar"
Botão azul no canto superior direito. Selecione "Nova implantação" (ou "Gerenciar implantações" se quiser atualizar uma existente).
2
Configure a implantação
Preencha as opções como na tabela abaixo:
CampoValor obrigatório
TipoAplicativo da Web
Executar comoEu mesmo (conta_institucional@gmail.com)
Quem pode acessarQualquer pessoa ← obrigatório para a API ser pública
Descrição11CMS API v2.0 (opcional, mas recomendado)
🚫 Se "Quem pode acessar" não for Qualquer pessoa, o site dará erro 403 ao tentar carregar os dados. Este campo é crítico.
3
Autorize as permissões
O Google pedirá para você autorizar o acesso. Clique em "Autorizar acesso", escolha a conta, e em "Avançado → Ir para o projeto". Na tela seguinte, clique "Permitir".
ℹ️ O alerta "App não verificado" é normal para projetos internos/institucionais. Não é um vírus — é apenas porque a Secretaria não pagou o processo de verificação formal do Google. Pode continuar.
4
Copie a URL da Web App
Após implantar, o Google exibirá a URL da API. Ela tem este formato:
🔗 https://script.google.com/macros/s/SEU_ID_AQUI/exec
Se for diferente da URL que já está no código do frontend, você precisará atualizar a constante API_URL no HTML do site.
Deploy · Passo 4 de 6

Executar o Setup da Planilha

Um único acesso à URL de setup cria automaticamente todas as 5 abas com seus cabeçalhos, dropdowns de validação e dados iniciais.

1
Acesse a URL de setup
Cole a URL abaixo no navegador (substituindo pela sua URL se for diferente):
🚀 https://script.google.com/macros/s/AKfycbzPh29ncgWef.../exec?action=setup
A resposta deve ser um JSON com "success": true e a descrição de cada aba criada.
2
Verifique a planilha
Abra a planilha. Você deve ver 5 abas na parte inferior: CONFIG, PRESENCA, DELEGADOS, AVISOS, FOTOS.
✓ A aba CONFIG já virá preenchida com os 13 eventos e todos com status aguardando. Você pode começar a gerenciar imediatamente.
3
Alternativa via menu
Você também pode executar o setup pela própria planilha: abra o menu "🏥 11ª CMS" → "⚙️ Configurar planilha (setup)". O menu aparece na barra de menus do Google Sheets após salvar o script.
Deploy · Passo 5 de 6

Publicar o Frontend no GitHub Pages

O site HTML fica hospedado no repositório cmsmoc/11conferencia do GitHub.

1
Acesse o repositório GitHub
🐙 https://github.com/cmsmoc/11conferencia
2
Substitua o arquivo HTML

🌐 https://cmsmoc.github.io/11conferencia
Deploy · Passo 6 de 6

Verificação Final do Sistema

Confirme que todos os componentes estão funcionando antes de divulgar o link.

  • API respondendo: Acesse [URL_API]?action=config no navegador e veja um JSON com "success": true e os 13 eventos.
  • Site carregando: Abra cmsmoc.github.io/11conferencia e veja o splash aparecer e desaparecer.
  • Cards dos eventos: Após o splash, os 13 cards dos eventos devem aparecer no grid.
  • Detalhe de evento: Clique em qualquer card — a tela de detalhe deve abrir com os botões de ação.
  • Avisos: Se houver avisos ativos com Ativo = SIM na aba AVISOS, o popup deve aparecer ao carregar o site.
  • Responsividade: Teste no celular ou use o DevTools (F12 → ícone de celular) para verificar mobile.
🎉

Sistema no ar!

Se todos os itens acima estão OK, o deploy foi bem-sucedido. A partir de agora, toda a gestão é feita diretamente na planilha Google — sem precisar mexer em código.

Manual de Gestão

Aba CONFIG — Gerenciar Eventos

Esta é a aba principal. Cada linha é um evento. As colunas editáveis controlam o que aparece no site.

🔒

Colunas travadas (A–F)

As colunas ID, Nome, Data, Horário, Local e Endereço têm proteção leve e não devem ser editadas — elas vêm do banco de dados mestre. Se o local de um evento mudar, use a coluna Aviso para comunicar a alteração ao público.

Colunas Editáveis
Coluna O que faz Valores aceitos Tipo
Status Define o estado do evento. Controla todos os botões do site automaticamente. aguardando / formulario_ativo / encerrada Obrigatório
Link Formulário URL do Google Forms de inscrição. Aparece como botão quando Status = formulario_ativo. URL completa do Forms Opcional
CSV Presença URL da planilha de presença publicada como CSV. Alimenta a lista de presença. URL …/pub?output=csv Opcional
Flyer URL URL direta da imagem do flyer (ibb.co, Drive, etc.). Exibe o painel lateral no detalhe. URL de imagem Opcional
Ata URL Link para o documento de ata/propostas. Ativa o botão de ata quando Status = encerrada. URL do Google Docs ou PDF Opcional
Aviso Texto de aviso exibido em destaque na página do evento. Use para mudanças de local, horário etc. Texto livre Opcional
Tipo Aviso Define se o campo Aviso é texto, imagem ou vídeo. texto / imagem / video Auto
Obs / Notas Informações adicionais exibidas na seção "Informações" da página do evento. Texto livre Opcional

🟢

O botão de inscrição aparece (desde que Link Formulário esteja preenchido). Badge verde "Ativo". Ative no dia anterior ao evento ou quando as inscrições abrirem.

encerrada → pós-evento

⚠️

ID_Conferencia precisa ser exato

O ID deve ser idêntico ao da coluna A da aba CONFIG. Diferença de maiúscula ou acento faz a filtragem falhar. IDs válidos: saude-mulher, grande-renascenca, lgbtqia, etc. Confira sempre antes de salvar.

Quando publicar?

Os delegados só aparecem no site quando o Status do evento na aba CONFIG for encerrada. Isso garante que ninguém veja uma lista parcial. Preencha a aba DELEGADOS e mude o status apenas quando a lista estiver completa e validada.

Aba AVISOS — Notificações e Alertas

Dois tipos de aviso controlam notificações no site: urgentes (popup + barra vermelha) e normais (barra de notícias).

🚨

Nível "urgente"

Exibe popup ao abrir o site. Após fechar, permanece como uma barra vermelha fixa no topo. Use para mudanças críticas (local, cancelamento).

📌

Nível "normal"

Aparece como cards de "últimas notícias" logo abaixo do hero, acima dos cards de eventos. Ideal para anúncios gerais ou atalhos para listas de presença recentes.

Como obter a URL direta do Google Drive

Clique com botão direito na imagem no Drive → "Obter link" → Certifique-se que o acesso é "Qualquer pessoa com o link". Então pegue o ID do link (a parte após /d/) e monte: https://drive.google.com/uc?export=view&id=SEU_ID. Alternativamente, use imgbb.com para hospedar e usar a URL direta.

Referência Técnica

Endpoints da API

Todos os endpoints são GET. A URL base é sempre a URL do Web App do Apps Script.

🔗 BASE: https://script.google.com/macros/s/AKfycbzPh29ncgWef.../exec
GET ?action=config
Retorna todos os eventos com campos booleanos já calculados no backend (podeInscrever, podeVerPresenca, etc.). Principal
Resposta
{ "success": true, "data": [ { "id": "saude-mulher", "status": "ENCERRADA", "podeInscrever": false, "podeVerPresenca": true, ... } ] }
GET ?action=presenca&id=saude-mulher
Retorna a lista de presença filtrada pelo ID do evento. Inclui metadados como total e contagem por segmento.
GET ?action=delegados&id=saude-mulher
Retorna delegados eleitos do evento especificado, com titulares e suplentes separados nos metadados.
GET ?action=avisos
Retorna todos os avisos com Ativo = SIM, separados por nível (urgente/normal) nos metadados.
GET ?action=fotos[&id=saude-mulher]
Retorna fotos com Incluir no Site = SIM. O parâmetro id é opcional para filtrar por evento.
GET ?action=setup
Cria/verifica todas as abas da planilha. Idempotente — pode ser chamado múltiplas vezes sem risco. Abas existentes com dados são preservadas. Setup único

Fluxo de Trabalho — Semana a Semana

Este é o roteiro operacional para a equipe da Secretaria Executiva durante o período das pré-conferências.

D-3
3 dias antes do evento
1. Abra a planilha → aba CONFIG → localize o evento.
2. Cole o Link Formulário do Google Forms de inscrição.
3. Mude o Status para formulario_ativo.
4. Se tiver flyer, cole a URL em Flyer URL.
5. Salve. O site atualiza automaticamente na próxima chamada à API.
✓ Não precisa publicar de novo nem mexer em código. A mudança na planilha já reflete no site.
D+0
No dia do evento
1. Se necessário, coloque um aviso na aba AVISOS (ex: mudança de sala).
2. Acompanhe a lista de presença conforme os participantes chegam.
D+1
1 dia após o evento
1. Preencha a aba PRESENÇA com os participantes do evento.
2. Preencha a aba DELEGADOS com os eleitos (se houver eleição).
3. Mude o Status na aba CONFIG para encerrada.
4. Os botões de presença e delegados aparecem automaticamente no site.
D+7
Após publicação da ata
1. Coloque o link da ata em Ata URL na aba CONFIG.
2. O botão de ata aparece automaticamente no site.
Suporte Técnico

Solução de Problemas

Diagnóstico e solução para os problemas mais comuns.

SintomaCausa provávelSolução
Site carrega mas não mostra os eventos API retornando erro ou planilha sem dados Acesse [URL_API]?action=config diretamente. Se erro, verifique as permissões da Web App. Se JSON mas vazio, execute ?action=setup.
Botão de inscrição não aparece Status ≠ formulario_ativo ou campo Link Formulário vazio Verifique ambos os campos na aba CONFIG. Os dois precisam estar corretos para o botão aparecer.
Lista de presença vazia ID da conferência na aba PRESENÇA diferente da aba CONFIG Compare os IDs. A coluna ID_Conferencia deve ser idêntica à coluna A da aba CONFIG, incluindo hífens e minúsculas.
Popup de aviso não aparece Coluna Ativo = NÃO ou Nível ≠ urgente Verifique as duas colunas na aba AVISOS. O popup só dispara para avisos com Ativo = SIM e Nível = urgente.
Erro 403 na API "Quem pode acessar" não está como "Qualquer pessoa" No Apps Script, vá em Implantar → Gerenciar implantações → edite a versão atual e altere o acesso para "Qualquer pessoa".
Fotos da galeria não carregam URL do Drive não tem permissão pública No Drive, clique com direito na imagem → "Obter link" → mude para "Qualquer pessoa com o link pode visualizar". Use a URL no formato uc?export=view&id=….
Mudei a planilha mas o site não atualizou Cache do navegador ou do Apps Script Force recarregamento com Ctrl+Shift+R. Se persistir, o Apps Script tem cache de ~5min. Aguarde ou crie uma nova versão da implantação.
"App não verificado" ao autorizar Projeto não passou pela verificação formal do Google Normal para projetos internos. Clique em "Avançado → Ir para o projeto" e depois "Permitir". Não é vírus.

Resposta saudável da API

Uma resposta correta sempre começa com {"success":true,"data":[…]}. Se o JSON começar com {"success":false,"error":"…"}, leia a mensagem de erro — ela indica exatamente o que está errado.
