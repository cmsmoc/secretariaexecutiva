1 — Paleta de Cores

O sistema usa 4 cores primárias. Cada ata recebe uma cor dominante aplicada ao thumbnail, stripe do card, header do modal e resumo executivo. As cores alternam entre as atas para criar ritmo visual na galeria.

Navy
#0D2E5A
Institucional · Header · Encaminhamentos
Blue
#1B6CB5
Interativo · Seções · CTAs
Green
#1E8A4A
Aprovações · Destaque positivo
Yellow
#F5C400
Acento · Eyebrows · Alerta
Regra de Alternância

Use a sequência: Navy → Green → Blue → Navy → Green → Blue... para atas consecutivas. Nunca use a mesma cor em duas atas consecutivas. A cor da 462ª é Navy mesmo que duas outras sejam Navy — o que importa é a sequência visual na galeria.

Atenção

O Yellow (#F5C400) nunca é usado como cor dominante de ata — apenas como acento. Com texto branco sobre amarelo a legibilidade é comprometida.

4 — Estrutura do Modal (Ata Completa)
Modal Header
Fundo na cor dominante da ata. Contém: eyebrow "CMSMOC", título "Nº Reunião Ordinária — CMSMOC", subtítulo com data · hora · local. Botão ✕ fixo no canto superior direito.
Resumo Executivo
Bloco visual na cor dominante. Contém: eyebrow com data, 3 chips de metadata (hora / local / presidente), 4 contadores, 4 cards de estatísticas, bloco de destaque, lista de temas tratados.
Síntese Label
Separador tipográfico: fundo off-white, borda superior em branco, borda inferior em navy 3px. Texto: 📄 Síntese em Montserrat 700 uppercase.
1. Expediente
Lista de ausências com .ausencia-item (nome + motivo). Separados por borda inferior.
2. Informes
Blocos .ata-pauta com borda esquerda azul. Cada pauta tem título e descrição.
3. Ordem do Dia
Título com classe .green. Cada item usa .ata-pauta. Votações usam .vote-box com variantes: .vote-aprovado, .vote-aprovado-unanime, .vote-abstenção.
4. Encaminhamentos
Lista com .ata-enc: número em badge navy + texto. Título com classe .yellow.
5. Encerramento
Texto corrido com horário de encerramento e responsável pelo fechamento da sessão.

6 — Prompt de Integração de Nova Ata

Envie este prompt junto com a transcrição processada e a convocação de cada nova reunião. O resultado deve ser integrado no arquivo HTML principal conforme o checklist abaixo.

Prompt Completo — Copie e use
Você é o assistente da Secretaria Executiva do CMSMOC (Conselho Municipal de Saúde de Montes Claros/MG).

Gere o bloco de dados para integração de uma nova ata no Módulo Atas do sistema digital do Conselho, seguindo EXATAMENTE a estrutura e padrão abaixo.

───────────────────────────────────────────────
DADOS DA REUNIÃO (preencha antes de enviar):
- Número: [ex: 464]
- Tipo: [Ordinária / Extraordinária]  
- Data: [ex: 07 de abril de 2026]
- Data curta: [ex: 07/04/2026]
- Hora: [ex: 13h30 – 17h00]
- Local: [ex: Plenário da Câmara Municipal]
- Presidente: [ex: Farley Sindeaux Ribeiro]
- Cor dominante: [Navy #0D2E5A / Blue #1B6CB5 / Green #1E8A4A]
  (alterne em relação à ata anterior: Navy → Green → Blue → repete)
───────────────────────────────────────────────
CONTEÚDO (cole abaixo):
[TRANSCRIÇÃO PROCESSADA ou RESUMO DA REUNIÃO]
[CONVOCAÇÃO se disponível]
───────────────────────────────────────────────

RETORNE exatamente neste formato JSON (sem markdown, sem comentários):

REGRAS para body_html:
- Use APENAS as classes CSS existentes: ata-section, ata-section-title, ata-pauta, ata-pauta-titulo, ata-pauta-desc, ata-enc, ata-enc-n, ata-enc-text, ausencia-item, ausencia-nome, ausencia-motivo, vote-box, vote-aprovado, vote-aprovado-unanime, vote-abstenção, ata-body-text
- Seções obrigatórias: 1.Expediente · 2.Comunicação e Informes · 3.Ordem do Dia · 4.Encaminhamentos
- Seção 3 usa classe "green" no título; Seção 4 usa classe "yellow"
- Votações usam .vote-box dentro do .ata-pauta-desc
- NÃO inclua o bloco de Resumo Executivo — ele é gerado automaticamente pelo JSON
7 — Checklist de Integração
1️⃣
Obtenha os dados da reunião — use o prompt acima com a transcrição processada + convocação. O retorno será um JSON completo.
2️⃣
Defina a cor dominante — alterne Navy → Green → Blue em relação à ata imediatamente anterior na galeria (ordem cronológica).
3️⃣
Adicione ao array ATAS no script Python gerador (ou diretamente no HTML, seguindo o padrão dos blocos existentes). Mantenha a ordem cronológica crescente.
4️⃣
Atualize o contador no header (header-badge-num) para o novo total de atas disponíveis.
5️⃣
Verifique visualmente — abra o HTML e confirme: thumbnail com ícone no canto, Resumo Executivo com 4 cards de stats, label "Síntese" antes do corpo textual.
6️⃣
Publique — substitua o arquivo no servidor/GitHub Pages. Nenhum outro arquivo precisa ser alterado.
