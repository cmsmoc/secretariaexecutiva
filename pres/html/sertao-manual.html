O que é o sistema

O SerTão Conselheiro é uma plataforma digital institucional para o Conselho Municipal de Saúde de Montes Claros (CMS-MOC). É uma PWA (Progressive Web App) — funciona como site, mas pode ser instalada como app no celular, funciona offline em modo cache, e envia notificações push.

O sistema foi desenvolvido inteiramente do zero, de forma incremental, sem custo operacional, usando infraestrutura gratuita (GitHub Pages + Google Apps Script + Google Sheets + Google Drive).

👥 Para os conselheiros
Acesso a documentos, atas, comissões, reuniões, notificações oficiais e solicitações formais — tudo no celular, com login individual.
🌐 Para o público
Acesso público aos documentos marcados como acesso=publico na planilha, sem necessidade de login. Transparência ativa.
🛠️ Para o admin (você)
Painel completo para gerenciar documentos, conselheiros, avisos, módulos, Drive sync e logs — sem precisar editar código.
📋 Para a Secretaria Executiva
Gestão de solicitações formais, envio de notificações push, controle de presenças e administração geral do Conselho.

🌐 Frontend — GitHub Pages
HTML + CSS + JS vanilla. PWA com service worker. Sem frameworks, sem build step. Deploy direto via git push para o repositório cmsmoc/doc.
⚙️ Backend — Google Apps Script
Web App publicado como doGet(). Recebe parâmetros GET, executa lógica, retorna JSON. Gratuito, sem servidor. URL fixa (nunca criar nova implantação, sempre "nova versão").
📊 Dados — Google Sheets
Planilha com 20 abas. Cada aba é uma "tabela". Índices de colunas mapeados em constantes no GAS (CC, CD, etc). ID: 1_6uxp2YjjmOKa5xkyc_l4jnVpCCex-M1GaXVSTn3HAM

Como uma requisição funciona (fluxo completo)

// 1. Frontend chama api.js
Api.getDocumentos()

// 2. api.js monta a URL
GET {GAS_URL}?action=getDocumentos&perfil=publico

// 3. GAS doGet() roteia para o handler
case 'getDocumentos': result = handleGetDocumentos(params)

// 4. Handler lê da planilha e filtra por acesso
_getSheet(ABA.DOCUMENTOS).getDataRange().getValues()

// 5. Retorna JSON
{ success: true, documentos: [...] }

// 6. api.js entrega para o componente da página
⚠️
Regra crítica #1: O GAS usa apenas GET via URL params. POST existe mas causa redirect/CORS. Nunca tentar enviar dados via POST direto — use sempre GET com params codificados.
⚠️
Regra crítica #2: A URL da implantação GAS é fixa. Ao atualizar o código: Implantar → Gerenciar implantações → lápis → Nova versão. Nunca "Nova implantação" (quebraria a URL).

Backend — Google Apps Script

Todo o backend está em um único arquivo Code.gs no Google Apps Script. Funções públicas (wrappers) são chamadas pelo Painel.html via google.script.run. O doGet() serve os endpoints para o frontend via URL.

Organização interna do Code.gs

Configuração global
_FALLBACK, _prop(), _getSS(), _getSheet(), ABA, índices de colunas (CC, CD…)
Cache & Log
_cacheGet/Set/Remove, _invalidarCache(), _logAction(), _flushLog() — log em buffer, gravado no finally
doGet() — roteador
Switch com 42 cases. Roteia action= para o handler correto. Try/catch global. _flushLog() no finally.
Handlers
Funções handle*(params) — lógica real. Wrappers globais (sem "handle") para google.script.run do Painel.
Setup
runSetup() cria todas as 20 abas se não existirem. seedComissoes() pré-carrega 9 comissões. Rodados uma única vez.
Drive Sync
handleSincronizarDrive() varre a pasta 1h8xwDb2V-6NG1c1PrgSfONjppfwSMT0c e importa arquivos novos como STATUS=pendente.

Perfis de acesso

Hierarquia numérica: quanto maior o número, mais acesso. Um perfil vê tudo que os menores veem.

const hierarquia = {
  admin:      5,   // você — acessa tudo + painel GAS
  secretaria: 4,   // acesso total exceto admin puro
  presidente: 3,   // redireciona para presidente.html
  diretoria:  2,   // redireciona para diretoria.html
  publico:    1,   // conselheiro padrão
};
💡
Para alterar o perfil de um conselheiro: planilha CONSELHEIROS → coluna PERFIL → alterar o valor. O próximo login dele já refletirá o novo perfil. Limpar cache do GAS após mudanças de perfil.

Painel Admin (Google Sheets sidebar)

Acesso: na planilha → menu CMS Docs Admin → Painel Admin. Abre modal lateral dentro do Sheets.

Ligar/desligar módulos (ativo | manutenção | inativo). Salva na aba CONFIG.
⚙️ Config
Editar textos da home, banners, configurações gerais. Salva via salvarConfigs().
📋 Log + Quotas
Log completo de acessos. Monitor de uso da API do GAS (limite: 20.000 req/dia).
⚠️
Se o Painel.html mostrar erro de sintaxe JavaScript, é porque a versão no GAS está desatualizada. Solução: abrir o Apps Script Editor → Painel.html → apagar tudo → colar conteúdo do Painel.html dos outputs.

Regras críticas — nunca quebrar

  • 🚨
    URL do GAS é fixa. Ao atualizar Code.gs: sempre "Nova versão" na implantação existente. Nunca criar implantação nova.
  • 🚨
    ID da planilha. Se a planilha for migrada, atualizar o fallback em _FALLBACK.SHEET_ID no Code.gs E nas Script Properties.
  • 🚨
    Colunas nas abas: só no final. Os índices CC.ID=0, CC.NOME=4 etc são hardcoded. Nunca inserir coluna no meio.
  • 🚨
    Chave de sessão. cms_session no localStorage. Não alterar — quebra login de todos os conselheiros logados.
  • 🚨
    Hierarquia de perfis. Os números (admin=5…publico=1) estão em _temAcesso() no GAS e na lógica do frontend. Não alterar sem revisar todos os pontos de uso.
  • 🚨
    SSH para push. Usar sempre git@github-cmsmoc:cmsmoc/doc.git com a chave ~/.ssh/id_cmsmoc.
  • ⚠️
    Cache do GAS. Após mudanças na planilha (perfis, documentos), rodar "Limpar Cache" no menu ou chamar _invalidarCache() — senão o GAS serve dados antigos por até 10 min.
