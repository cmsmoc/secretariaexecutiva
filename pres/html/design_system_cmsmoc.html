Princípio de Navegação

Cada módulo é um arquivo HTML independente hospedado em GitHub Pages. A navegação entre módulos usa href direto para a URL do módulo. A percepção de sistema único vem da identidade visual compartilhada — não de uma arquitetura técnica acoplada. Isso é intencional: simplicidade operacional máxima.

05
Regras do Sistema
✓ Faça
Use sempre a stripe tricolor (Yellow+Green+Blue) no rodapé do header — é a assinatura visual imutável do sistema.
✓ Faça
Alterne cores dominantes entre atas/documentos consecutivos para criar ritmo visual na galeria.
✓ Faça
Use Montserrat 900 para números e títulos principais — o peso extremo é parte da identidade.
✓ Faça
Mantenha o ícone CMSMOC saindo pelo canto inferior direito dos thumbnails — sem filtro de cor.
✓ Faça
Use fundos off-white (#F4F6F9) para o corpo de página em módulos em light mode.
✗ Evite
Não use Yellow (#F5C400) como cor dominante de módulo — apenas como acento. Falta contraste com texto branco.
✗ Evite
Não crie gradientes entre cores não-adjacentes no sistema (ex: Navy direto para Yellow — use sempre uma cor intermediária).
✗ Evite
Não altere as cores primárias por "tons" — o sistema não admite versões clareadas ou escurecidas das 4 cores base.
✗ Evite
Não use bordas arredondadas acima de 18px em componentes de card — o sistema é refinado, não infantilizado.
✗ Evite
Não misture as fontes de sistema (Inter, Roboto, Arial) com Montserrat/Open Sans em nenhum componente.
06
Espaçamento e Ritmo
O sistema usa uma escala de 4px como base. Os valores mais usados:
4px — micro (gap entre ícone e texto)
8px — pequeno (gap chips, tags)
12px — base (padding interno de badges)
20px — médio (padding de cards)
24px — confortável (gap de galeria mobile)
36px — generoso (padding interno de modal)
48px — seção (padding vertical de galeria)
